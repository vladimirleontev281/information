Весьма доходчиво описано как создать репозиторий и "запушить" файлы (первые 2 шага).

Теперь нюансы:
  • Название репозитория.

    Где-то там по тексту написано следующее: "Важно, чтобы название репозитория было в виде «username.github.io», где username — имя вашего аккаунта на Гитхабе". Так вот не принципиально. Мой, скажем, называется "information".
  • Настройка GitHub Pages.

    Когда я первый раз попытался захостить головную страницу и поступил так как сказано в 3-м шаге статьи, меня ожидала следующая картина:



    Начал разбираться и оказалось, что требуется настроить GitHub Pages. Для этого, находясь в своем репозитории, нужно выбрать настройки ...



    ... скролом долистать до GitHub Pages и выставить первый параметр в позицию "master branch".



  • Ссылка на ресурс.

    После того как параметр будет выставлен, появится рабочая ссылка на корень репозитория. Именно её нужно использовать для просмотра результатов и для ссылок на картинки и шрифты (конечно учитывая дальнейшую вложенность).



    В моем случае корневой будет ссылка https://vladimirleontev281.github.io/information/. Эту страницу я положил в папку For_GitHub. Соответственно, чтоб ее просмотреть иду по ссылке https://vladimirleontev281.github.io/information/For_GitHub/For_GitHub.html

    Так же и при подключенни шрифтов:
    src: url("https://vladimirleontev281.github.io/information/ и далее по вложенностям")
